# Stylesheet Internationalization

> Version: 1.0.0
> Last Updated: 2026-03-29

Conventions for LTR/RTL support and internationalization in Pavillion stylesheets.

## Established Convention

Pavillion supports multilingual content including RTL languages. All directional CSS must use logical properties.

### Logical Properties Mapping

| Physical (avoid) | Logical (use) |
|-------------------|---------------|
| `margin-left` | `margin-inline-start` |
| `margin-right` | `margin-inline-end` |
| `padding-left` | `padding-inline-start` |
| `padding-right` | `padding-inline-end` |
| `left` | `inset-inline-start` |
| `right` | `inset-inline-end` |
| `text-align: left` | `text-align: start` |
| `text-align: right` | `text-align: end` |
| `border-left` | `border-inline-start` |
| `border-right` | `border-inline-end` |
| `float: left` | `float: inline-start` |
| `float: right` | `float: inline-end` |

### Shorthand Properties

```scss
// Symmetric padding/margin (safe - no directionality)
padding: var(--pav-space-4);
margin: var(--pav-space-2) var(--pav-space-4);

// Inline/block shorthands (preferred for directional)
padding-inline: var(--pav-space-4);
margin-block: var(--pav-space-2);
```

## Anti-Patterns

### Physical Direction Properties

```scss
// BAD: physical properties break in RTL
.sidebar {
  margin-left: var(--pav-space-4);
  padding-right: var(--pav-space-2);
  text-align: left;
  border-left: var(--pav-border-width-2) solid var(--pav-border-primary);
}

// GOOD: logical properties work in both directions
.sidebar {
  margin-inline-start: var(--pav-space-4);
  padding-inline-end: var(--pav-space-2);
  text-align: start;
  border-inline-start: var(--pav-border-width-2) solid var(--pav-border-primary);
}
```

### Physical Positioning

```scss
// BAD: physical position values
.icon {
  position: absolute;
  left: var(--pav-space-2);
}

// GOOD: logical position values
.icon {
  position: absolute;
  inset-inline-start: var(--pav-space-2);
}
```

## Exceptions

These properties don't require logical equivalents:

- `top`, `bottom` -- vertical axis is not affected by text direction
- `margin-top`, `margin-bottom` -- block direction rarely changes; `margin-block-start`/`margin-block-end` are preferred but physical forms are acceptable
- `gap` -- not directional
- Transforms using `translateX` -- may need manual RTL handling; this is acceptable
- Symmetric shorthand properties (`padding: var(--pav-space-4)`, `margin: 0 auto`) -- no directionality

## Known Drift

- Many older components use physical properties (`margin-left`, `padding-right`, `text-align: left`)
- Some components use `left`/`right` for absolute positioning
- Migration to logical properties is ongoing
