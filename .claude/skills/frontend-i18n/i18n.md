# Frontend Internationalization

Use i18next with i18next-vue for all user-facing text. Organize translations by feature namespace for maintainability.

## Namespace Organization

One JSON file per feature area:

```
src/client/locales/
├── en/
│   ├── system.json         # Navigation, common UI elements
│   ├── authentication.json # Login, password reset
│   ├── calendars.json      # Calendar management
│   ├── event_editor.json   # Event creation/editing
│   └── ...
└── es/
    ├── system.json
    └── ...
```

## Component Usage

Use `useTranslation` with namespace and optional keyPrefix:

```vue
<script setup>
import { useTranslation } from 'i18next-vue';

// Load namespace with key prefix
const { t } = useTranslation('event_editor', {
  keyPrefix: 'editor',
});
</script>

<template>
  <button>{{ t('create_button') }}</button>
  <!-- Resolves to: event_editor.editor.create_button -->
</template>
```

## Translation Key Structure

Use nested objects for logical grouping:

```json
{
  "editor": {
    "create_button": "Create Event",
    "update_button": "Update Event",
    "error_no_calendar": "Please select a calendar"
  },
  "categories": {
    "loading_categories": "Loading categories..."
  }
}
```

## Rules

- Always use namespace + keyPrefix — avoids key collisions
- Keep keys descriptive: `error_no_calendar` not `error1`
- Group related keys under common prefixes
- Fallback language is English (`fallbackLng: 'en'`)
